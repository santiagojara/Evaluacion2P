FLUJO DE TRABAJO DE GITFLOW
  • Main (o Master): Es la rama principal y estable del proyecto. Representa la producción en vivo del software.
  • Develop: Es la rama donde se integran todas las características que se han completado y se han probado.
  • Feature/: Estas son ramas temporales que se crean desde develop. Se utilizan para desarrollar nuevas características o funcionalidades específicas.
  • Hotfix/: Estas son ramas temporales que se crean desde main (o master). Se utilizan para corregir problemas críticos o errores en producción que necesitan ser solucionados de inmediato.

FLUJO DE PROCESO GESTIÓN DE CAMBIO
  • 1. Se crea una rama develop a partir de main.
  • 2. Se crea una rama release a partir de la rama develop.
  • 3. Se crean ramas feature a partir de la rama develop.
  • 4. Cuando se termina una rama feature, se fusiona en la rama develop.
  • 5. Cuando la rama release está lista, se fusiona en las ramas develop y main.
  • 6. Si se detecta un problema en main, se crea una rama hotfix a partir de main.
  • 7. Una vez terminada la rama hotfix, esta se fusiona tanto en develop como en main.
