PROCESO ACS
  • Definición de Objetivos y Alcance: Establecer los objetivos específicos que el proceso ACS busca alcanzar y delimitar su alcance dentro del proyecto o sistema.
  • Análisis y Diseño: Examinar los requisitos y diseñar las estrategias para cumplir con las metas del proceso ACS.
  • Implementación: Desarrollar y poner en práctica las soluciones y estrategias diseñadas en la etapa anterior.
  • Evaluación y Pruebas: Revisar y probar el proceso ACS para asegurar que cumple con los requisitos y objetivos establecidos.
  • Mantenimiento y Mejora Continua: Realizar seguimiento continuo y aplicar mejoras al proceso ACS basándose en los resultados de la evaluación y feedback recibido.

FLUJO DE TRABAJO DE GITFLOW
  • Master: Representa la rama de producción. Contiene el código que ha sido probado y se considera estable.
  • Develop: Rama de integración para el desarrollo en curso. Se utiliza como base para crear nuevas características y realizar pruebas de integración.
  • Feature: Se utilizan para desarrollar nuevas características para el próximo lanzamiento.
  • Release: Prepara el código para un nuevo lanzamiento, realiza pruebas finales y corrige problemas menores.
  • Hotfix: Rama destinada a corregir errores críticos en la rama Master.

FLUJO DE PROCESO GESTIÓN DE CAMBIO
  • Identificación del Cambio: Registro y evaluación de cambios propuestos, considerando la importancia y urgencia.
  • Evaluación y Análisis: Evaluación de la viabilidad y el impacto del cambio, incluyendo análisis de impacto, riesgos y complejidad.
  • Aprobación: Desarrollo e implementación del cambio en el código fuente o configuración del sistema.
  • Pruebas: Realización de pruebas para garantizar que el cambio cumpla con requisitos funcionales y no cause regresiones.
  • Despliegue: Implementación del cambio en el entorno de producción, seguido de monitoreo post-implementación.
  • Documentación: Actualización de documentación técnica y de usuario para reflejar el cambio realizado.
  • Cierre: Finalización formal del proceso, incluyendo el archivo de documentación y la revisión para identificar mejoras.
